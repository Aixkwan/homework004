因为git的分支必须指向一个commit，没有任何commit就没有任何分支 提交第一个commit后git自动创建master分支”

提交（git commit）一次后，后面新建的分支不管有没有再用该新分支提交数据，git branch都会显示

提交到远程git仓库才能用 git branch 查看到分支(远程分支) git branch -a 可查看到创建后但没提交到远程git仓库的本地分支

如果要将其他分支合并到主分支上

那就先切换到主分支上 再 git merge ***(分支名)

将相应的分支合并到主分支上

然后在主分支上提交文件到远程git仓库才会在远程git仓库上显示合并过来的项目文件(相当于更新)

相当于每个组员有对应的分支(可直接在远程git仓库上创建)

也可以通过命令面板创建(git branch checkout -b 分支名) 但需在此分支上提交文件到远程git仓库才能被查询到
